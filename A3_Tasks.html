  1. Ask user to insert two integers

  1. Prompt user to insert
    1. A word
    2. A character
  2. Find if the character exists in the word. Possible prints below.

A3_T5 Temperature converter (TEST TASK)

Create a temperature unit conversion program.

Start the program by listing options for the user:

  1. Celsius to Fahrenheit
  2. Fahrenheit to Celsius
  3. Exit

Prompt user to insert choice. After the decision to convert, ask the amount of current temperature (use the floating point datatype). Lastly show the converted value to the user.

For the unit conversions, use the formula Celsius = (Fahrenheit - 32) / 1.8

Data representation examples:

  • 50.0 °F
  • 10.0 °C

If the user chooses option Exit, notify the user: Exiting...

Use 1 decimal precision to round the converted value.

Example program runs

Program starting.

Options:
1 - Celsius to Fahrenheit
2 - Fahrenheit to Celsius
0 - Exit
Your choice: 1
Insert the amount of Celsius: 23

Program ending.



Program starting.

Options:
1 - Celsius to Fahrenheit
2 - Fahrenheit to Celsius
0 - Exit
Your choice: 2
Insert the amount of Fahrenheit: 100
100.0 °F equals to 37.8 °C

Program ending.



Program starting.

Options:
1 - Celsius to Fahrenheit
2 - Fahrenheit to Celsius
0 - Exit
Your choice: 1
Insert the amount of Celsius: -15
-15.0 °C equals to 5.0 °F

Program ending.

Note! version 2.0.1 allows both Celsius and Celcius. The syntax in the grading tool is covered with [cs] regex pattern.

A3_T7 Decision trees

In this task the idea is to create a program where user can define an integer and choose the decision structure being applied to the inserted integer. Certain rules must be applied to specified value tresholds and the decision structure has partial role in the end results. Structure, order and operators matter, so do exactly as the task describes.

  1. Prompt user to insert value as an integer.
  2. Display menu
    1. option 1 - In one multi-branched decision
    2. option 2 - Independent if-statement decisions
    3. option 0 - Exit
  3. Prompt user to choose option
  4. Apply following math operations in the options 1 & 2
    1. One multi-branched decision structure:
      1. Value is 400 or more => add 44 to the existing value
      2. Value is 200 or more => add 22 to the existing value
      3. Value is 100 or more => add 11 to the existing value
    2. Independent if-statement decisions one after another:
      1. Value is 400 or more => add 44 to the existing value
      2. Value is 200 or more => add 22 to the existing value
      3. Value is 100 or more => add 11 to the existing value
    3. Exit: “Exiting…”
  5. At the end of the options 1 & 2, show the results like in the example program runs.

Other possible output variats:

  • “Unknown option.”
